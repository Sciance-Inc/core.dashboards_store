## Bases de données

La base de données `paie` doit être liée au projet. Veuillez vous référer à la section [Lier une base de données](/using/configuration/databases) pour plus d'informations sur la façon de lier une base de données.

## Seed

::alert{type=warning} 1 seed sont requises pour le fonctionnement du tableau de bord. ::

Le tableau de bord nécessite les 3 seeds suivantes : 

### etat_empl : 
Seed obligatoire. La liste des états d'emplois du systèmes de paie. Vous pouvez retrouver l'information dans la table : PAI_TAB_ETAT_EMPL de votre système de paie. Pour le rapport de démission, nous avons ajouté une colonne pour indiquer, soit par un 0 si ce n'est pas un état de démission ou un 1 si l'état d'emploi est une démission
Endroit de la seed : seeds/marts/human_resources/schema.yml Champs obligatoires :

 1. etat_empl : Code unique de la qualification
 1. descr : Description du code de qualification
 1. etat_actif : Indique si l’employé est actif
 1. empl_retr : Indique si l’employé est retraité
 1. empl_cong : Indique si l’employé est en congé
 1. cong_lt : Indique si l’employé est en congé de longue durée
 1. mpl_resi : Indique si l'employé a démissionné


```bash
dbt seed --full-refresh
```

## Marts

Les marts suivants doivent être activés pour que le tableau de bord fonctionne. Veuillez vous référer à la section [Activer un mart](/using/configuration/enabling) pour plus d'informations sur la façon d'activer un mart.

- `human_resources`

## Dbt project specification
> Update your `cssxx.dashboards_store/dbt_project.yml` file with the following snippet.

```yaml
# cssxx.dashboards_store/dbt_project.yml
models: 
  store:
    marts:
      human_resources:
        +enabled: True  
    dashboards: 
      resignation:
        +enabled: True
    interfaces:
      paie:
        +enabled: True
```
